[cron:introspection-weekly]

Weekly csagent introspection (Wave B1). Follow skill introspection-ops.

1. Read `<stateDir>/logs/runs.jsonl` — last 7 days (status, error_kind, duration_ms, tokens).
2. `memory_search` wing episodic + keyword search for failures, cron, gateway, digest.
3. Optional read-only: `memory_get user-profile.niksh`, `memory_get agent-profile.composer` (meta) — compare to friction; never write meta wing.
4. Write one proposal note `introspection-YYYY-MM-DD` via `memory_save` (wing default).

Required sections: Summary, Failures, Friction, Suggested notes, Suggested issues.

When Friction includes communication or agent-behavior items (ambiguous prompts, ADVICE/DO confusion, Telegram length/delivery, tone): also include **User-profile patch (proposal)** and **Agent-profile patch (proposal)** with Add / Update / Remove / Source evidence tables. Mark pending approval — never auto-merge to meta profiles.

Skeleton: deploy/prompts/introspection-weekly.template.md

No code changes, no cron edits, no git. Proposals only.
