TParser **daily digest** — single agent (no delegates). Produce the full daily digest in ONE run.
Window: **last 24 hours** — compute `WINDOW_START = now − 24h` (UTC), `WINDOW_END = now`. **Do not use /api/feed.**

You do the whole job yourself: fetch once, bucket into five topics, analyze, then output the final Telegram digest (Russian). Output **ONLY** the final Telegram message body — no preamble, no tool narration.

## Setup (once)

1. `memory_get` name=`tparser-workflow` — tag/sphere rules and interests.
2. Fetch posts **once**:
   - `GET http://127.0.0.1:8002/api/posts/recent-live?limit=300`
   - Paginate `after_rowid` until posts are older than `WINDOW_START`.
   - Keep items with `created_at` (or `date`) >= `WINDOW_START`.
3. Hydrate: batch `GET /api/posts/by-keys?keys=channel_id:post_id,...` (max 50 keys per call).
4. Bucket each post into **one** topic by `topic_tags` (else summary keywords). A post belongs to at most one topic (highest-priority match).

## Topics (five buckets)

- **AI / ML / LLM** — AI, ML, LLM, benchmark, research, agent, RAG, fine-tuning
- **AISec / MLSec** — AISec, MLSec, LLM security, adversarial, jailbreak, prompt injection, model abuse
- **InfoSec / AppSec** — InfoSec, AppSec, security, vulnerability, CVE, exploit, malware, incident
- **Programming / devtools** — programming, devtools, language, framework, open source, library, IDE
- **DevSecOps / DevOps** — DevSecOps, DevOps, CI/CD, K8s, kubernetes, cloud, infra, SRE, terraform

## Per-post analysis (within each topic, priority desc)

Analyze **all** matched posts per topic. For each post keep this block:

```

- Bullets preserving the gist, key claims, named entities, and **all URLs** from `urls`/message (external links verbatim).

**Вердикт агента:**
2–4 предложения: что это значит в разрезе текущих интересов (tparser-workflow). Если ссылок несколько — какая главная и почему.

**Telegram:** https://t.me/c/{short}/{post_id}
**Источник:** {best external URL or «(нет URL)»}
```

Link rules: `short` = str(channel_id) with leading `-100` stripped. Never `t.me/@channel`. Pick the most relevant external URL for Источник; list others under Структура.

**Deep check** (optional, ≤1 per post): for a post with external http(s) link and thin summary, use `browser_navigate` + `browser_snapshot` (browser-ops) on the **one** primary URL, or `GET /api/posts/{post_id}/analysis?channel_id=`. Never invent content.

**Dedup:** window `WINDOW_START`–`WINDOW_END` only; do **not** write `memory_fact_add` / `seen_post` facts. No Obsidian writes.
