TParser bi-hourly digest (last 2 hours). **Do not use /api/feed.**

1) Call memory_get name=tparser-workflow for tag/sphere rules.
2) Compute WINDOW_START = now UTC minus 2 hours (ISO8601).
3) Fetch ingested posts:
   - GET http://127.0.0.1:8002/api/posts/recent-live?limit=150
   - Paginate with after_rowid if needed until items are older than WINDOW_START.
   - Keep items where created_at (or date) >= WINDOW_START.
4) Hydrate cards: batch GET /api/posts/by-keys?keys=channel_id:post_id,... (max 50 keys per request).
5) Filter **by topic_tags** (primary) and summary when tags sparse. Match tags/keywords from tparser-workflow spheres: AI, ML, LLM, MLSec, security, InfoSec, AppSec, AISec, DevSecOps, programming, devtools, benchmark, vulnerability, LLM security. Include General News if tags match.
6) Include **ALL** matching posts in the window (no top-N cap). Skip posts already in seen_post facts (memory_fact_query subject=seen_post) if duplicate.
7) For each included post resolve links (Hermes-compatible):
   - **tg_link** — deep link to THIS post in Telegram (not the channel chat):
     `https://t.me/c/{short}/{post_id}` where short = str(channel_id) with leading "-100" stripped
     (e.g. channel_id=-1001534155873, post_id=22699 → https://t.me/c/1534155873/22699).
   - **source_url** — first http(s) URL from `urls` (pipe-separated) that is NOT t.me / telegram.me;
     if `urls` empty or only Telegram chat links, extract http(s) links from `message`;
     if still none → literal `(нет URL)`.
   - Never use t.me/@channel or t.me/channel_name as tg_link; always t.me/c/…/post_id.
8) memory_fact_add subject=seen_post predicate=channel_id object=post_id for each included post.
9) **Output ONLY the Telegram message body** (Russian, Hermes hot-digest style — compact lines with links):
   Header: `📬 TParser · последние 2ч · {count} постов · {window_end_utc}`
   Blank line, then one line per post:
   `**{title}** — {1–2 sentences: what happened + why it matters}. Ценно для {tag/sphere}. {source_url} — {tg_link}`
   - title: short headline (channel name or gist of summary), not raw post_id.
   - Use GET /api/posts/{post_id}/analysis?channel_id= only when summary is thin.
   - End with stats line: `{raw_ingested} / {tag_matched} / {sent}`
   If tag_matched=0: output exactly `📬 TParser · последние 2ч · новых релевантных постов нет.`
10) Do not invent posts or URLs. Do not re-analyze. Do not write Obsidian files (evening job only).
