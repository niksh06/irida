TParser bi-hourly digest (last 2 hours). **Do not use /api/feed.**

**Goal:** A short Telegram digest readable in under 2 minutes. **Do NOT list every matching post** — use the output template in `deploy/prompts/tparser-bi-hourly-output-template.txt` (summary + top by priority only).

1) Call memory_get name=tparser-workflow for tag/sphere rules.
2) Compute WINDOW_START = now UTC minus 2 hours (ISO8601); WINDOW_END = now UTC.
3) Fetch ingested posts:
   - GET http://127.0.0.1:8002/api/posts/recent-live?limit=150
   - Paginate with after_rowid if needed until items are older than WINDOW_START.
   - Keep items where created_at (or date) >= WINDOW_START.
4) Hydrate cards: batch GET /api/posts/by-keys?keys=channel_id:post_id,... (max 50 keys per request).
5) Filter **by topic_tags** (primary) and summary when tags sparse. Match tags/keywords from tparser-workflow spheres: AI, ML, LLM, MLSec, security, InfoSec, AppSec, AISec, DevSecOps, programming, devtools, benchmark, vulnerability, LLM security. Include General News only if tags match.
6) Rank **all** tag-matched posts in the window by `priority` (desc). Let MATCHED = count.
   - TOP_N = 15 (Telegram body). If MATCHED < 15, use MATCHED.
   - **Dedup:** only posts with `created_at` inside the 2h window; do **not** write `memory_fact_add` / `seen_post` facts.
7) Links (Hermes-compatible) for each post you mention in top-N:
   - **tg_link** — `https://t.me/c/{short}/{post_id}` where short = str(channel_id) with leading "-100" stripped.
   - **source_url** — first external http(s) from `urls` or `message`; if none → `(нет URL)`.
   - Never use t.me/@channel as tg_link.
8) **Output ONLY the Telegram message body** (Russian). **Hard limit ~3500 characters** (prefer one message). Follow this structure exactly:

   Header:
   `📬 TParser · 2ч · {WINDOW_END}`
   `{MATCHED} релевантных из {raw_ingested} · топ-{TOP_N}`

   `## Сводка`
   - Totals: relevant count, analysis breakdown (completed / none) from hydrated fields.
   - `### По категориям` — markdown table, top categories by count (max 7 rows).
   - `### Каналы (топ-8)` — one line: `Name (n), …`.

   `---`

   `## Топ-{TOP_N} по priority`
   For rank 1..TOP_N only:
   `**{rank}.** `{priority}` · **{channel_or_title}** · {primary_tag/sphere}`
   `{one sentence: what happened + why it matters}.`
   `{tg_link}` on its own line (no markdown link syntax).

   Optional (only if space left, max ~600 chars):
   `## Темы дня (кратко)` — 3 subsections (Безопасность/AI Security, AI/разработка, DevSecOps/инфра) with 2–4 one-line bullets each; **no duplicate tg_links** already in top-N.

   Footer:
   `---`
   `Не вошли {MATCHED − TOP_N} постов. В чате: «топ-50», «только InfoSec», «только AI»; полный день — GET /api/posts/by-date-range.`
   Stats: `{raw_ingested} / {MATCHED} / {TOP_N}`

   If MATCHED=0: output exactly `📬 TParser · последние 2ч · новых релевантных постов нет.`

9) Style rules:
    - No wall of one-line-per-post (old Hermes hot-digest list).
    - No raw post_id in titles; use channel name or short gist.
    - Use GET /api/posts/{post_id}/analysis?channel_id= only when summary is thin.
    - Do not invent posts or URLs. Do not re-analyze. Do not write Obsidian files (evening job only).
