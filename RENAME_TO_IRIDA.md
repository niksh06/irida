# Переименование: `csagent` → **Irida**

> Статус: **proposal / planning** · Дата: 2026-06-19 · Автор обсуждения: NK + ассистент
> Этот файл — план и обоснование. Кода он не меняет. Исполнять только по явной отмашке.

---

## TL;DR

Агент перестаёт быть «просто Cursor Agent»: к основному рантайму (Cursor SDK) добавляется
**резервный движок на Anthropic Claude Agent SDK**. Это меняет суть продукта — теперь это
**локальный персональный агент с двумя движками и фолбэком**, а не обёртка над одним SDK.
Под новую суть выбрано имя **Irida** (греч. Ирида — богиня-вестница, радуга-мост между мирами):
звучит красиво, продолжает Hermes-линию, а namespace в нише AI-агентов — чистый.

Переименование = новый CLI-бинарь `irida`, домашняя папка `~/.irida`, env-префикс `IRIDA_*`,
плюс миграция состояния и временный shim для старого `CSAGENT_*`.

---

## 1. Предыстория

- Проект родился как **`csagent`** — *local-first personal agent powered by the Cursor SDK*,
  с **Hermes-inspired UX** (sessions, skills, MCP, safety) и принципиальным решением:
  **без второго model/провайдер/tool-loop** — работу исполняет родной agent-runtime Cursor.
- Вокруг него выросла инфраструктура: домашняя директория `~/.csagent` (logs, `runs.jsonl`,
  профиль, `.agent`), gateway (`deploy/gateway-keep.json`), фоновые задачи через launchd/cron,
  набор npm-скриптов (`build`, `dev`, `start`, `doctor`, `run:agent`, `chat`, `tui`, `sessions`,
  `eval`, `smoke`, …), env-переменные `CSAGENT_*`.
- В коде закрепился префикс `CSAGENT_` и хелпер `csagentHome()` (`src/env.ts`), на который
  завязаны `mcpServers.ts`, `doctorChecks.ts`, `cronJobs.ts`, `gateway_cmd.ts`,
  `gatewayStatus.ts`, `memorySiloOps.ts` и др.

## 2. Что изменилось — триггер переименования

Рассматривается добавление **второго, резервного рантайма** на базе **Claude Agent SDK**
(ранее «Claude Code SDK») — встраиваемого агентного движка Anthropic (тот же харнесс, что
питает Claude Code: agent loop, тулы, MCP, сабагенты, hooks, permission-режимы, компакция;
TS-пакет `@anthropic-ai/claude-agent-sdk`, Python `claude-agent-sdk`).

Идея: **основной движок — Cursor SDK, резервный — Claude Agent SDK** (failover / выбор движка).
Это прямо расширяет исходную философию «без второго loop» — поэтому продукт концептуально
становится **другим агентом**, и имя «Cursor Agent / csagent» перестаёт его описывать
(и привязывает бренд к одному из двух провайдеров).

> ⚠️ **Открытый архитектурный вопрос (не блокирует переименование).** Резерв можно сделать
> двумя способами:
> - **(A)** полноценный второй рантайм на **Claude Agent SDK** — максимум возможностей, но это
>   буквально второй agent loop (расходится с исходным «без второго loop»);
> - **(B)** только **Messages API** (`@anthropic-ai/sdk`) как *способность* внутри текущего
>   Cursor-loop — Claude дёргается точечно, одна петля сохраняется (ближе к исходной философии).
>
> Имя **Irida** подходит обоим вариантам. Решение (A) vs (B) фиксируется отдельно (см. §8).

## 3. Почему именно переименование (а не «оставить как есть»)

1. **Семантика.** Имя должно описывать суть: «двухдвижковый агент с фолбэком», а не один SDK.
2. **Нейтральность бренда.** «Cursor Agent» жёстко привязывает к одному из двух провайдеров,
   хотя теперь их два равноправных (основной + резерв).
3. **Позиционирование.** NK строит для себя, но не исключает заработок — значит имя должно быть
   уникальным и не тонуть среди тёзок.

## 4. Журнал выбора имени (что обсуждали)

Концепция, под которую искали имя: *локальный персональный агент, основной движок + резерв,
не падает, переключается между мирами*. Грани: **двойственность / чередование**,
**отказоустойчивость / возрождение**, **мост между мирами**, **оборотень (смена движка)**.

| Кандидат | Вердикт | Причина |
|---|---|---|
| **Loki** | ❌ | (1) Коллизия в домене — **Grafana Loki** (логи), тот же dev/devops-рынок. (2) Бог обмана/предательства — тонально против «надёжный агент с резервом». Хорош был только образ оборотня. |
| **Iris** | ❌ | Звучание и смысл отличные (вестница + радуга-мост), но имя **плотно занято ровно в AI-агентах**: `mrgehlot/iris-agent` (Python agent framework), `intersystems-community/iris-agentic-dev` (MCP-сервер для Claude Code), InterSystems IRIS Agent, Iris-AgenticAI, ColumnTax Iris. Точное совпадение + та же ниша. |
| **Bifrost** | ❌ | Красивое «мост между мирами», но занято **в той же концепции**: `maximhq/bifrost` — активный AI-gateway (мульти-провайдер, MCP, фолбэк/роутинг), плюс `ming86/bifrost-llm-gateway`. |
| **Proteus** | ⏳ | Оборотень («protean = многоликий»), чистая коннотация. Запасной вариант, если ядро идеи — «смена формы». |
| **Pollux / Dioscuri** | ⏳ | Близнецы, что **по очереди** делят бессмертие — идеальная метафора primary/fallback. Сильный запасной. |
| **Bennu / Simurgh / Huma** | ⏳ | «Возрождение / никогда не падает» (резерв). Красивые, редкие. |
| **Irida** | ✅ | **Выбрано.** Полная греческая форма Iris: тот же миф (вестница + радуга-мост = два рантайма), та же красота звучания, но **в нише AI-агентов namespace чист**. |

## 5. Что значит Irida (бренд)

**Ирида** — греческая богиня-вестница и олицетворение **радуги**, соединяющей мир богов и людей.
Идеально ложится на концепцию:
- **мост между двумя мирами** = два рантайма (Cursor ↔ Claude Agent SDK);
- **вестница** = продолжение Hermes-линии исходного UX;
- бонус: «iris» — радужка глаза → лёгкая отсылка «агент, который видит» (security-вектор NK).

Одна строка для README: *Irida — local-first personal agent: один интерфейс, два движка, мост
между ними.*

## 6. Карта точек переименования (`csagent` → `irida`)

| Слой | Сейчас | Станет | Файлы/места |
|---|---|---|---|
| CLI-команда / пакет | `csagent`(?) | `irida` | `package.json` (`name`, `bin`), shell-completions |
| Домашняя папка | `~/.csagent` | `~/.irida` | runtime-путь + миграция данных |
| Хелпер пути | `csagentHome()` | `iridaHome()` | `src/env.ts` (+ все импортёры) |
| Env-переменные | `CSAGENT_HOME`, `CSAGENT_DATABASE_URL`, `CSAGENT_SECRETS_KEY`, `CSAGENT_TEST_PG_URL`, … | `IRIDA_*` | `src/env.ts`, `mcpServers.ts`, `doctorChecks.ts`, тесты |
| Permission-правила | вшитые `export CSAGENT_*`, `Read(~/.csagent/**)` | `IRIDA_*`, `Read(~/.irida/**)` | `.claude/settings.local.json` |
| Gateway | метки/конфиг `csagent` | `irida` | `deploy/gateway-keep.json`, `gateway_cmd.ts`, `gatewayStatus.ts` |
| Сервисы | launchd/cron-метки | `irida` | LaunchAgents plist, crontab, `cronJobs.ts` |
| Доки | ссылки на `~/.csagent`, `csagentHome()` | новое имя | `README.md`, `CLAUDE.md` |

> Точные текущие значения `name`/`bin` в `package.json` и метки сервисов — **подтвердить перед
> правкой** (в обсуждении мы видели только `CSAGENT_*`, `csagentHome()` и `~/.csagent`).

## 7. Пошаговый план миграции

**Принцип:** один координированный проход в отдельной ветке, со shim'ом обратной совместимости,
чтобы ничего не отвалилось «на лету» и не потерялось состояние.

1. **Подготовка**
   - [ ] Ветка `rename/irida`; зафиксировать чистый `git status`.
   - [ ] Инвентаризация вхождений: `rg -n 'csagent|CSAGENT|\.csagent|csagentHome'` — сверить с §6.
2. **Код: env-слой с обратной совместимостью**
   - [ ] В `src/env.ts` ввести `iridaHome()` и чтение `IRIDA_*` **с фолбэком на legacy `CSAGENT_*`**
         (читаем оба, пишем только новые; на чтение старого — `console.warn` deprecation).
   - [ ] `csagentHome()` оставить тонким алиасом → `iridaHome()` на один релиз, затем удалить.
3. **Код: остальные модули**
   - [ ] Прогнать переименование идентификаторов/строк по импортёрам (`mcpServers`, `doctor`,
         `cron`, `gateway*`, `memorySiloOps`).
4. **Домашняя директория: миграция, не переезд вслепую**
   - [ ] При старте: если есть `~/.csagent` и нет `~/.irida` — **скопировать/перенести**
         (logs, `runs.jsonl`, профиль, `.agent`), записать `migrated-from-csagent` маркер.
   - [ ] Не удалять `~/.csagent` автоматически — оставить до подтверждения пользователем.
5. **CLI / пакет**
   - [ ] `package.json`: `name`/`bin` → `irida` (при желании сохранить `csagent` как deprecated-alias bin).
   - [ ] Обновить completions и подсказки.
6. **Сервисы и gateway**
   - [ ] Переименовать launchd/cron-метки и `gateway-keep.json`; предусмотреть разовый
         un-load старой метки → load новой (иначе будут крутиться два сервиса).
7. **Permissions / .claude**
   - [ ] В `.claude/settings.local.json` заменить `CSAGENT_*` export-правила и `Read(~/.csagent/**)`
         на `IRIDA_*` / `Read(~/.irida/**)` (старые можно оставить временно — не мешают).
8. **Доки**
   - [ ] `README.md` + `CLAUDE.md`: новое имя, `~/.irida`, `iridaHome()`, env `IRIDA_*`.
9. **Проверка**
   - [ ] `npm run typecheck && npm test && npm run doctor` — зелёные.
   - [ ] `npm run smoke` на чистом профиле и на мигрированном из `~/.csagent`.
   - [ ] Убедиться, что запущен **один** сервис, а не дубль.
10. **Зачистка (следующий релиз)**
    - [ ] Удалить legacy `CSAGENT_*` shim, deprecated bin-alias, старую метку сервиса.
    - [ ] Предложить пользователю удалить `~/.csagent`.

## 8. Решения, которые ещё предстоит зафиксировать

- [x] **(A) vs (B) — РЕШЕНО (2026-06-19): вариант A — выбор движка на стороне пользователя.**
      Пользователь сам выбирает рантайм из «деки» — **Cursor SDK** или **Claude (Agent) SDK**,
      что ему удобнее/нравится. То есть два полноценных движка на выбор, переключаемых
      пользователем; единого «дефолтного» движка нет. Это сознательный отход от исходного
      «без второго loop» — ради свободы выбора. (Имя продукта от этого не зависит — **Irida**.)
- [ ] Сохранять ли `csagent` как deprecated-alias CLI (плавный переход) или резать сразу.
- [ ] Публиковать ли в npm (тогда нужен `npm view irida` и, возможно, scoped-имя `@nk/irida`).
- [ ] Финальная формулировка слогана/описания для README.

## 9. Риски / на что обратить внимание

- **Потеря состояния** при переезде `~/.csagent` → `~/.irida` — отсюда шаг 4 (миграция, не rename).
- **Дубль сервисов** launchd/cron, если старую метку не выгрузить — отсюда шаг 6.
- **Секреты в env** (`CSAGENT_SECRETS_KEY` и пр.) — при переименовании префикса не «уронить» их,
  поэтому фолбэк-чтение старого префикса на переходный период (шаг 2).
- **Внешние интеграции**, ожидающие `CSAGENT_*` — shim покрывает, но стоит поискать в других репо.
